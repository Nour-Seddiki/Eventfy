Project: Build a complete multi-page React web application called "Saha·Event" — a premium Algerian wedding hall (salle des fêtes) booking platform. Use Next.js App Router, Tailwind CSS, shadcn/ui. The app has two user roles: Client and Owner (Locateur).

🎨 DESIGN SYSTEM — READ THIS FIRST

Aesthetic direction: "Classic Modern Luxury" — NOT generic AI SaaS. Think: a high-end Parisian atelier meets Algerian architectural heritage. The design must feel handcrafted, editorial, and physically tangible.

3D & Depth effects (critical, use throughout):

Cards must have real CSS transform-style: preserve-3d hover tilt effect (mouse-tracking 3D card tilt on venue cards)

Hero section has a layered parallax depth with floating 3D geometric shapes (gold wireframe octahedron, rotating slowly)

Buttons have a pressed/emboss effect using layered box-shadows that simulate physical depth

Section transitions use perspective transforms — content slides in with a 3D rotate on scroll

Venue gallery uses a 3D carousel (CSS 3D transform rotateY) not a flat slider

The booking widget card has a thick cast shadow making it look physically elevated off the page

Use backdrop-filter: blur + layered rgba backgrounds to create true glass depth (not flat glassmorphism)

Color palette (strict — no purple, no generic blue):

--gold: #B8953F (muted antique gold, NOT bright yellow)

--gold-light: #D4AF6A

--cream: #F7F2EA

--parchment: #EDE4D3

--espresso: #18120E

--bark: #2E2218

--stone: #6B5B45

--sage: #4A6741 (success states)

--terracotta: #B54A38 (error/cancel states)

--ink: #1C1712

Typography (import from Google Fonts):

Display/Headings: "Cormorant" weight 300–700, italic variants

UI/Body: "Jost" weight 300–600

Monospace accents (prices, dates, codes): "DM Mono"

NEVER use Inter, Roboto, or system fonts

Texture & detail:

Subtle noise grain overlay on all dark sections (SVG feTurbulence filter, 3% opacity)

Thin 1px gold rule lines used as section dividers (not full-width, 60px centered)

Arabesque corner ornaments (SVG) on hero and modal headers

All rounded corners: border-radius: 2px for inputs/buttons (sharp, architectural), 12px for cards, 0 for nav

🗓️ REAL BOOKING SYSTEM — UPDATE ALL CALENDAR COMPONENTS

This is the core availability engine. Apply to ALL calendar instances across the app.

Availability Rules:

A date becomes unavailable when:

A client has a confirmed reservation on that date (auto-blocked by system)

A client has a pending reservation on that date (shown as "En attente" — soft block, different color)

The owner manually blocks that date

A date becomes available again when:

A reservation is cancelled by owner or client

The owner unblocks a manually blocked date

Calendar Color Legend (apply consistently everywhere):

⬜ Cream/white = Available — clickable

🟡 Gold tint bg = Pending (reservation awaiting owner confirmation) — not clickable, shows tooltip "En attente de confirmation"

▪️ Espresso dark bg = Confirmed booking — not clickable, shows tooltip "Réservé"

🔲 Dark bark bg + lock icon = Blocked by owner — not clickable, shows tooltip "Indisponible"

⬛ Light stone bg = Past date — not clickable

🟦 Gold border = Today

🟨 Gold bg = Selected date (client picking)

Calendar interactions on Venue Detail Page (client view):

Hovering an available date shows tooltip: "Disponible — Cliquez pour sélectionner"

Clicking a date selects it and auto-fills the booking widget start date

If client tries to click unavailable date: gentle shake animation + tooltip explaining why

Multi-day selection: click start date then end date, range highlights in gold

If selected range includes any unavailable day: show error "Ces dates incluent un jour déjà réservé. Veuillez ajuster votre sélection."

Owner Calendar (in Owner Dashboard — Calendrier tab):

Full-width month calendar with 3 modes (toggle buttons top right):

🚨 EDGE CASES & VALIDATION — IMPLEMENT THESE STATES

Double-booking prevention:

When client submits a booking request for dates that just got confirmed by another client simultaneously → system detects conflict → shows error: "Ces dates viennent d'être réservées. Veuillez choisir d'autres dates." → calendar refreshes

Pending soft-block:

When a pending request exists for a date → that date shows as "En attente" (gold tint) on calendar

Other clients can still request the same dates (first confirmed wins) — show warning: "⚠ Une demande est en cours pour ces dates. Votre réservation sera confirmée si la précédente est refusée."

Owner deletes venue:

All pending reservations auto-cancelled with client notification

Confirmed future reservations: owner must cancel individually with mandatory reason + client notification

KYC expired / rejected after previous approval:

User sees banner: "⚠ Votre vérification a expiré. Veuillez soumettre à nouveau vos documents."

Booking capabilities suspended until re-verified

CCP Receipt validation:

Client uploads receipt → owner sees it in dashboard
